# 算法笔记

## 1. 算法

### 1.1 二分查找

---

#### 是什么

在**有序顺序表**中每次取中间位置比较，根据大小舍弃一半，直到找到或区间为空。

```
[1, 3, 5, 7, 9, 11, 13] 找 7

mid = ⌊(0+6)/2⌋ = 3 → arr[3]=7 命中，2 次
```

**前提**：①有序 ②可随机访问。链表不能二分。

**时间复杂度**：O(log n)。n=10⁶ 只需约 20 次比较，n 翻倍只多一步。

---

#### 为什么快

| n | 顺序查找（最坏） | 二分查找（最坏） |
|---|---------------|---------------|
| 100 | 100 | 7 |
| 10,000 | 10,000 | 14 |
| 1,000,000 | 1,000,000 | 20 |

本质：每次砍一半，砍 log₂n 次只剩一个。

---

#### 判定树

查找过程可画成平衡二叉树：

```
                   arr[3]=7
                  /        \
           arr[1]=3      arr[5]=11
           /    \         /     \
       arr[0]=1 arr[2]=5 arr[4]=9 arr[6]=13
```

- 内部结点 = 成功查找 = n 个
- 外部结点 = 失败查找 = n+1 个
- 深度 = ⌊log₂n⌋ + 1
- ASL(成功) ≈ log₂(n+1) - 1

---

#### 标准代码（精确查找）

```python
def binary_search(arr, x):
    low, high = 0, len(arr) - 1
    while low <= high:
        mid = (low + high) // 2
        if arr[mid] == x:   return mid
        elif arr[mid] < x:  low = mid + 1
        else:               high = mid - 1
    return -1
```

---

#### 变体：找边界

```python
# 第一个 >= x 的位置（lower_bound）
def lower_bound(arr, x):
    low, high = 0, len(arr)
    while low < high:
        mid = (low + high) // 2
        if arr[mid] >= x:
            high = mid      # 往左压缩
        else:
            low = mid + 1
    return low

# 第一个 > x 的位置（upper_bound）
def upper_bound(arr, x):
    low, high = 0, len(arr)
    while low < high:
        mid = (low + high) // 2
        if arr[mid] > x:
            high = mid
        else:
            low = mid + 1
    return low
```

**关键区别**：精确查找 `while low <= high` + `high = mid - 1`；找边界 `while low < high` + `high = mid`。混用会死循环。

---

#### 二分答案

问题有单调性（"最大值最小"类），猜 mid、用 check(mid) 验证：

```python
low, high = 0, max_val
while low < high:
    mid = (low + high) // 2
    if check(mid):
        high = mid      # 可行，尝试更小
    else:
        low = mid + 1   # 不可行，必须更大
print(low)
```

典型：砍树、分饼干、跳石头、木材仓库。

---

#### 易错

1. 精确查找 `low <= high`，找边界 `low < high`
2. `high = mid` 时必须 `low < high`，否则死循环
3. 前提：有序 + 顺序表，链表不能用
4. 判定树深度 ⌊log₂n⌋+1，不是 log₂n

### 1.2 前缀和

---

#### 是什么

预处理数组，`pref[i]` 表示前 i 个元素的和。**任意区间和 O(1) 查询**。

```
原数组 arr: [3, 1, 4, 1, 5, 9]
前缀和 pref: [0, 3, 4, 8, 9, 14, 23]
             ↑ pref[0]=0（空区间）

区间 [l, r] 的和 = pref[r+1] - pref[l]
例：[1,3] 即 1+4+1=6 → pref[4]-pref[1]=9-3=6
```

---

#### 为什么需要

暴力求区间和每次 O(n)，m 次查询 O(nm)。前缀和建表 O(n)，查询 O(1)。

| | 暴力 | 前缀和 |
|--|------|--------|
| 建表 | 无 | O(n) |
| 单次区间查询 | O(n) | O(1) |
| m 次区间查询 | O(nm) | O(n+m) |

---

#### 一维前缀和

```python
# 建表（0-indexed）
pref = [0] * (n + 1)
for i in range(n):
    pref[i + 1] = pref[i] + arr[i]

# 查询区间 [l, r]（0-indexed，闭区间）
def range_sum(l, r):
    return pref[r + 1] - pref[l]
```

---

#### 二维前缀和

`pref[i][j]` = 从 (0,0) 到 (i-1, j-1) 的矩形和。

```python
# 建表
pref = [[0] * (m + 1) for _ in range(n + 1)]
for i in range(n):
    for j in range(m):
        pref[i+1][j+1] = pref[i][j+1] + pref[i+1][j] - pref[i][j] + arr[i][j]

# 查询矩形 (r1,c1) 到 (r2,c2)
def rect_sum(r1, c1, r2, c2):
    return (pref[r2+1][c2+1] - pref[r1][c2+1]
            - pref[r2+1][c1] + pref[r1][c1])
```

---

#### 差分（反向应用）

差分是前缀和的逆：`diff[i] = arr[i] - arr[i-1]`。**区间修改 O(1)**。

```python
# 对区间 [l, r] 每个元素 +val
diff[l] += val
diff[r + 1] -= val

# 还原原数组：对 diff 做前缀和
for i in range(1, n):
    diff[i] += diff[i - 1]
```

---

#### 技巧

- `pref` 长度 n+1，下标 0 存 0，方便处理从 0 开始的区间
- 区间 `[l, r]` 长度 = r - l + 1，和 = `pref[r+1] - pref[l]`
- 前缀和也可以用哈希表：`pref[i] - pref[j] = target` 转化为两数之差问题

---

### 1.3 差分（Difference Array）

---

#### 是什么

给数组的某个区间 `[l, r]` 每个元素都 +1，暴力需 O(区间长)。差分只在两端做标记，最后前缀和还原，O(1) 完成区间修改。

**本质**：前缀和的逆运算。

---

#### 原理

```
区间 [l, r] 统一 +val：

diff[l] += val        ← 从 l 开始生效
diff[r+1] -= val      ← 从 r+1 开始失效

对 diff 做前缀和 → 原数组每个位置的实际变化值
```

---

#### 例子

```
原数组全 0，给 [2, 5] 加 3：

diff:  [0, 0, 0, 0, 0, 0, 0, 0]
       l=2 +3 → diff[2] += 3
       r=5 -3 → diff[6] -= 3

diff:  [0, 0, 3, 0, 0, 0,-3, 0]

前缀和还原：
[0, 0, 3, 3, 3, 3, 0, 0]
      ↑  ↑  ↑  ↑
      2  3  4  5  → 全部 +3
```

---

#### 代码模板

```python
n = 200000  # 数组长度
diff = [0] * (n + 2)   # 多开两个位置防越界

# 区间 [l, r] 每个元素 +val
diff[l] += val
diff[r + 1] -= val

# 对所有区间标记完后，前缀和还原
for i in range(1, n + 1):
    diff[i] += diff[i - 1]

# diff[i] 就是位置 i 的最终值
```

---

#### 多次区间修改

```python
# 配方 1: [2,4]  配方 2: [3,5]  配方 3: [1,6]

diff[2]+=1, diff[5]-=1   # 配方1
diff[3]+=1, diff[6]-=1   # 配方2
diff[1]+=1, diff[7]-=1   # 配方3

前缀和后 → 每个位置的值 = 覆盖该位置的配方数
```

多个区间叠加时，diff 数组上直接累加标记，最后统一还原。

---

#### 差分 vs 暴力

| | 暴力 | 差分 |
|--|------|------|
| 一个区间修改 | O(区间长) | O(1) |
| n 个区间修改 | O(n × 区间长) | O(n) |
| 查询单点值 | O(1) | O(1)（前缀和还原后） |

---

#### 典型应用

- 区间统一加值 → 最后询问每个位置的值
- 统计每个位置被多少个区间覆盖（咖啡题、浇水题）
- 配合前缀和实现"区间修改 + 区间查询"

### 1.4 哈希表（散列表）

---

#### 是什么

一种"根据键直接访问值"的数据结构。核心：**键 → 散列函数 → 地址 → 取值**。

类比：学号后两位 → 打开对应柜子 → 取档案。不用从第一个翻到最后一个。

增删改查**平均 O(1)**，最坏 O(n)（全冲突）。

---

#### 为什么需要

| | 数组 | 链表 | 哈希表 |
|--|------|------|--------|
| 查找学号 20240001 | 开 20240001 个位置，炸 | 从头查到尾 O(n) | 直达 O(1) |
| 统计字符频率 | 下标只能整数 | 每次 O(n) | 字符 → 频率 O(1) |

**空间换时间**：多花内存建表，查找从 O(n) 降到 O(1)。

---

#### 散列函数

##### 除留余数法（考试唯一考）

`H(key) = key % p`，p 取 ≤ 表长的**质数**。

p 取质数能减少扎堆：key 全是偶数时 mod 10 只用 5 个槽，mod 11 全用。

| 其他方法 | 公式 | 场景 |
|---------|------|------|
| 直接定址 | H(key)=a·key+b | 范围小的连续整数 |
| 数字分析 | 取分布均匀的位 | 已知 key 分布 |
| 平方取中 | key² 中间几位 | 随机性好 |

---

#### 装填因子 α

`α = n / m`（记录数 ÷ 槽位数）。**α 决定效率，和 n 多大无关。**

- α=0.1 → 很空，基本不冲突
- α=0.8 → 拥挤，查找变慢
- α→1 → 接近退化 O(n)

---

#### 冲突处理

##### 开放定址法

冲突找下一个空位：`Hi = (H(key) + di) % m`

| 方式 | di 序列 | 特点 |
|------|--------|------|
| 线性探测 | 1, 2, 3... | 简单，易聚集 |
| 平方探测 | ±1², ±2²... | 减少聚集 |
| 双散列 | i×H₂(key) | 最均匀 |

**线性探测聚集现象**：冲突元素连续挤在一起，后续查找路径越来越长。

**删不了！** 直接删会断探测链，只能打标记（懒惰删除）。

##### 链地址法（拉链法）

每个槽位挂链表，冲突的串一起。α 可 >1，删除随便。

---

#### ASL 手算（必考）

逐个插入，记录比较次数，求均值。

```
H(key)=key%13，线性探测

19(1次)→14(1)→23(1)→01(2)→68(1)→20(1)→84(3)→27(4)→55(3)→11(1)

ASL = (1+1+1+2+1+1+3+4+3+1)/10 = 1.8
```

公式：链地址 ≈ 1+α/2；线性探测 ≈ ½(1+1/(1-α))

---

#### Python 模板

```python
# 频率统计
freq = {}
for x in nums:
    freq[x] = freq.get(x, 0) + 1

# 去重 / 存在性
seen = set()
if x in seen:  # O(1)

# 两数之和 / A-B=C
d = {}
for i, x in enumerate(nums):
    if target - x in d:
        ans += d[target - x]
    d[x] = d.get(x, 0) + 1
```

---

#### 易错

- 平均 O(1)，最坏 O(n)
- 开放定址不能直接删
- ASL 看 α 不看 n
- Python 的 dict/set 直接就是哈希表

### 1.5 DFS 递归枚举（回溯）

---

#### 是什么

一种"把所有可能试一遍"的方法。每个决策点分叉出几个选择，深度优先走到底，算出结果，然后回退试别的选择。

类比：走迷宫，每条岔路先钻到底，不通再退回上一个岔路口试另一条。

---

#### 代码骨架

```python
def dfs(当前走到第几步, 当前状态):
    if 走到头了:
        记录/更新答案
        return            # 退回上一步

    for 每种选择:
        做选择            # 更新状态
        dfs(下一步, 新状态)  # 往深处走
        撤销选择          # 恢复状态，试下一个
```

---

#### 例题：Perket（选配料）

每种配料选/不选，求酸度积与苦度和的最小绝对差。

```python
def dfs(i, sour, bitter, cnt):
    if i == n:                # 每种都决策完了
        if cnt > 0:           # 至少选了1种
            ans = min(ans, abs(sour - bitter))
        return                # 退回

    # 分支1：选
    dfs(i + 1, sour * s[i], bitter + b[i], cnt + 1)
    # 分支2：不选
    dfs(i + 1, sour, bitter, cnt)
```

```
调用树（n=2）：

dfs(0,1,0,0)
  ├─ 选0  → dfs(1,s[0],b[0],1)
  │        ├─ 选1  → dfs(2,...,...,2) → 算
  │        └─ 不选1 → dfs(2,...,...,1) → 算
  └─ 不选0 → dfs(1,1,0,0)
           ├─ 选1  → dfs(2,...,...,1) → 算
           └─ 不选1 → dfs(2,1,0,0) → cnt=0 跳过
```

---

#### 适用场景

- 子集枚举（选/不选每种） — Perket
- 全排列 — 八皇后
- 组合 — 选 k 个数
- 路径搜索 — 迷宫、棋盘

---

#### 和 DP 的区别

| | DFS 回溯 | DP |
|--|---------|-----|
| 方向 | 自顶向下 | 自底向上 |
| 状态 | 一路带参数走下去 | 存在数组里查表 |
| 适合 | n 小（≤20） | n 可以大 |
| 实现 | 递归 | 循环填表 |

### 1.6 记忆化搜索

---

#### 是什么

把递归函数的结果存起来，下次同样的参数直接返回，避免重复计算。

```
递归：算过一次忘记了，下次再算一遍
记忆化：算过就记在表里，下次直接查，O(1)
```

---

#### 为什么需要

```
斐波那契 fib(5)：

裸递归：
fib(5) = fib(4) + fib(3)
      = fib(3)+fib(2) + fib(2)+fib(1)
      = ... fib(1) 被算了 4 次！

记忆化：
算完 fib(3) 存起来，下次直接取，不再递归。
```

---

#### 代码模板

```python
# 斐波那契
memo = {}
def fib(n):
    if n <= 1:
        return n
    if n in memo:           # 算过直接取
        return memo[n]
    memo[n] = fib(n-1) + fib(n-2)
    return memo[n]
```

---

#### 三维例子（P1464 Function）

参数范围 0~20，用三维数组预存：

```python
dp = [[[1] * 21 for _ in range(21)] for _ in range(21)]

for a in range(21):
    for b in range(21):
        for c in range(21):
            if a == 0 or b == 0 or c == 0:
                dp[a][b][c] = 1
            elif a < b and b < c:
                dp[a][b][c] = dp[a][b][c-1] + dp[a][b-1][c-1] - dp[a][b-1][c]
            else:
                dp[a][b][c] = dp[a-1][b][c] + dp[a-1][b-1][c] + dp[a-1][b][c-1] - dp[a-1][b-1][c-1]

# 查询直接 dp[a][b][c]，O(1)
```

---

#### 适用场景

- 递归函数，参数范围有限
- 同一组参数会被反复调用
- 参数是整数，可以当数组下标

---

#### 和 DP 的关系

```
记忆化搜索 = 自顶向下（递归 + 存结果）
DP 递推   = 自底向上（先算小的，逐步推到大的）

同一个问题，两种写法。P1464 用 DP 递推更方便。
```

### 2.2 Python sys 模块 — 快速 I/O

---

#### `input()` 为什么慢

`input()` 每次调用都要：读一行 → 去掉换行 → 编码处理 → 返回字符串。10⁶ 次调用积累的开销足以超时。

---

#### 三个加速层级

```python
import sys

# ① 单行快读（比 input() 快 2-3 倍）
line = sys.stdin.readline()

# ② 全部读完再处理（比 input() 快 5-10 倍）
data = sys.stdin.buffer.read().split()  # 返回 bytes 列表

# ③ 直接转成整数数组（最快）
data = list(map(int, sys.stdin.buffer.read().split()))
```

---

#### 使用方式：指针遍历

```python
import sys

data = list(map(int, sys.stdin.buffer.read().split()))
p = 0
T = data[p]; p += 1

for _ in range(T):
    n = data[p]; p += 1
    # ... 按顺序取 data[p], data[p+1], ...

# 输出
sys.stdout.write('结果')
```

---

#### 为什么 `read().split()` 这么快

`input()` = 读一行 → 处理 → 返回 → 循环再读一行 → 再处理……

`read().split()` = **一口气全读进内存** → 一次性切好。后续用 `data[p]` 直接取数组元素，零 I/O 开销。

---

#### 适用场景

| 数据量 | 用 |
|--------|-----|
| < 10⁴ | `input()` 够用 |
| 10⁴ ~ 10⁵ | `sys.stdin.readline()` |
| > 10⁵ | `sys.stdin.buffer.read().split()` |
